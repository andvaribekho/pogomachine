GOAL
Split main.js (~5357 lines) into focused modules. No gameplay logic,
numbers, or behavior changes at any point. Every step leaves the game
fully playable.

THE CORE PROBLEM: SHARED MUTABLE STATE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Every system in main.js reads and writes the same flat module-level
variables: ball, ballVelocity, enemies[], platforms[], scene, world, etc.
You cannot split into files without making that sharing explicit first.

Solution: create src/core/context.js that exports all shared mutable
state as a single object. Every new module imports what it needs from
context.js and mutates it directly. main.js initializes the Three.js
objects (scene, camera, ball, …) and writes them into context.js on
startup.

context.js exports:
  - Three.js objects: scene, world, camera, renderer, pillar, ball,
      shieldMesh  (assigned by main.js at init time, not created here)
  - state object with all scalar game variables:
      ballVelocity, gravity, terminalVelocity, stompImpulse,
      bounceVelocity, score, hp, coins, ammo, maxAmmo, combo,
      impulseMode, controlMode, twistBMode, isPaused, isGameOver,
      isLevelComplete, currentLevel, platformsPassedThisLevel,
      piercingBulletsUnlocked, vampiricLifeUnlocked, comboShieldUnlocked,
      hasShield, invulnerabilityTimer, damageCooldown, timeScale,
      … (all ~50 current flat let variables)
  - arrays: platforms[], bullets[], enemies[], particles[],
      floatingTexts[], crates[], goldBlocks[], coinPickups[], cannons[],
      shockwaves[], pillarLaserRings[], bounceCubes[], pillarSpikes[],
      spikeTraps[], sawBlades[], floaters[], acidPuddles[]
  - drag object  { active, x, targetRotation }
  - touchPointerIds Set
  - platformBandIndex Map
  - shakeOffset, cameraBasePos  (reused Vector3s for camera shake)

Note: src/core/state.js (the unused createGameState factory) describes
this same state. Use it as reference when writing context.js, then
delete it or keep it as documentation.

  function animate(_time, frame) {
    const realDt = min(clock.getDelta(), 0.033)

    if (arModeActive) {
      updateArMode(realDt, frame)
      renderer.render(arScene, camera)
      return
    }

    updateGrayscaleBackground(realDt)

    const dt = realDt * state.timeScale
    state.scaledTime += dt

    // world rotation lerp
    if (!state.isPaused) { … }

    if (!state.isGameOver && !state.isPaused && !state.isLevelComplete) {
      applyGravityStep(dt)
      updateShooting(dt)
      updatePowerups(dt)
      scene.updateMatrixWorld(true)
      checkBallCrateHit(previousY)
      checkBallGoldBlockStomp(previousY)
      handlePlatformUndersideCollision(previousY)
      handlePlatformCollision(previousY)
      updateSpikeTraps(dt)
      updatePillarSpikes(dt)
      updateSawBlades(dt)
      updatePillarLaserRings(dt)
      updateBullets(dt)
      updateEnemies(dt)
      updateFloaters(dt)
      updateAcidPuddles(dt)
      updateShockwaves(dt)
      updateCannons(dt)
      updateGoldBlocks(dt)
      updateFallingCratesAndGold(dt)
      updateCoinPickups(dt)
      updateParticles(dt)
      updateBounceCubes(dt)
      recyclePlatforms()
      updateFloatingTexts(dt)
      // combo sprite follow
      updateTileFlashes(dt)
      updateCamera(dt)
    }

    updateBallVisual()
    renderer.render(scene, camera)
  }

Final main.js target: ~250 lines.

CROSS-CUTTING CONCERNS TO HANDLE CAREFULLY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

reloadAmmo()
  Called by enemies (stomp kills), hazards (cannon stomp), and platform
  collision. Lives in bullets.js. Enemies and hazards import it.

spawnBounceCubes()
  Called by killEnemyAt() and damageGoldBlock(). Lives in pickups.js.
  enemies.js imports it from there.

detachGroundEnemiesFromTile / detachAcidPuddlesFromTile
  Called by platforms.js when breaking tiles.
  These functions live in enemies.js.
  platforms.js imports enemies.js — one-way dependency, no cycle.

clearTower()
  Calls clearBullets (bullets.js), clearEnemiesAndParticles (enemies.js),
  clearCoinPickups, clearAcidPuddles, deactivateAllBounceCubes (pickups.js),
  plus iterates platforms/floaters/laserRings/pillarSpikes/sawBlades.
  Lives in platforms.js, imports the clear functions from each module.

twistBMode
  Used in createBat, createSpikedBall, createFloatingEnemyWithOptions to
  choose scene vs world parent. Accessed via context.state.twistBMode.

shopTilePlat / shopTileRef / shopUsed
  Written in createPlatform, read in handlePlatformCollision and shop.js.
  All in context.state — no special handling needed.
